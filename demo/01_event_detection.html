Load data¶

The time series with gauging station data should already have been check and/or preprocessed. The timeseries must contain:

  • an index in datetime format and easily converted to a DatetimeIndex.
  • a streamflow columns with streamflow values in $m^3\ s^{-1}$
  • a suspended_sediment column with suspended sediment concentrations (SSC) in $mg\ l^{-1}$

The time series file can contain additional columns, but there will be ignored when loading the data with the hysevt.events.metrics.get_gauge_data function. This is the same function that is used to load the time series for event detection, so first check that it is loaded correctly.

The resolution of the time series should be sub-daily. Depending on the temporal resolution of the time series the search window may have to be adjusted (see Detect events below).

Detect events¶

The event detection is done with the hysevt.events.detection.hydro_sediment_events function. A number of arguments and parameters need to be specified for it, and some are optional. Take a look at the documentation:

The first part of the detection is the identification of hydrological events with local-minima hydrograph separation. For this a search window has to be specified for the local-minima algorithm.

The station data in this example is in hours, so we need to specify a local-minima search window in hours. In catchments where melt processes are import a window of about a day is reasonable. Note that the detection routine allows for longer event durations than the specified window, but the shortest possible event duration is half the length of the detection window.

In [6]:
days = 1 # number of days in local minima detection window
window = (24 * days) - 3 # search window in hours, just below the secified number of days

Next we can specify filters for the hydrological events detected. It is not uncommon to have missing values, but we don't want to keep events with too much missing data. So lets remove events where we are missing more than 20 % of either streamflow or SSC data. If you wish you can specify a separate value for streamflow and SSC.

In [7]:
max_missing=0.2

The next filter criteria relates to event magnitude. The local-minima hydrograph separation simply splits your time series into event sized bits. But depending on what you're interested in, you might want to remove the small events. This can be done by specifying a peak SSC threshold. If an events maximum SSC values (i.e. peak) is below this threshold it will be filtered out. You can specify this theshold in two ways:

  1. use a quantile threshold from you whole station time series with the SSC_threshold_by_quantile agrument
  2. set a fixed threshold yourself with SSC_threshold, in $mg\ l^{-1}$. NB! this will override the quantile threshold!

In this example we'll use the 0.9 quantile as the SSC threshold:

In [8]:
filter_SSC_quantile = 0.9

For convenience we'll use this to specify the event detection version, which gets coded into the event ID numbers together with the station ID. This will be the first part of the event ID tag assigned to each event.

SSC peak threshold sensitivity¶

As you can see from the log file, the 0.9-quantile threshold removes quite a few events. Let's have a look at the sensitvity of the detection routine depending on the peak SSC threshold.

In [16]:
# load the final events
final_events = pd.read_csv(ODIR.joinpath("hydro_sediment_events_list.csv"))
# load the events that were removed because they were too small
small_SSC_events = pd.read_csv(
    ODIR.joinpath("events_small_magnitude.csv")
)  
# merge the two back together and extract their event time series
non_filtered_events = pd.concat([final_events, small_SSC_events])
non_filtered_event_series = hysevt.events.metrics.get_event_series(
    non_filtered_events, gauge_data
)
# get the peak SSC values of each event
non_filtered_event_SSC_peaks = np.array(
    [event.suspended_sediment.max() for event in non_filtered_event_series]
)
In [17]:
# quantiles 
quantiles = gauge_data.suspended_sediment.dropna().quantile(
    [0.5, 0.6, 0.7, 0.75, 0.8, 0.85, 0.9, 0.95, 0.99]
)

# create a dataframe with each quantile threshold
threshold_sensitivity = quantiles.to_frame()
threshold_sensitivity["final_events"] = [
    (non_filtered_event_SSC_peaks > q).sum() for q in quantiles
]
threshold_sensitivity["dropped_events"] = (
    len(non_filtered_events) - threshold_sensitivity.final_events
)
threshold_sensitivity[
    "final_events_fraction"
] = threshold_sensitivity.final_events / len(non_filtered_events)
